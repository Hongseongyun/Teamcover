## 01. IA (Information Architecture)

Teamcover 서비스의 **정보 구조(IA)**를 정의하는 문서입니다.  
메뉴/페이지/기능 간 관계를 한눈에 볼 수 있도록 정리합니다.

### 1. 최상위 메뉴 구조

- **예시**
  - 대시보드
  - 회원 관리
  - 경기/리그 관리
  - 정산/회비
  - 공지/알림
  - 설정(클럽/계정)

> 실제 서비스에서 사용 중인 메뉴명을 기준으로 아래 표를 채워 주세요.

### 2. 페이지/화면 목록

| ID    | 메뉴      | 화면명         | 경로(URL)      | 설명                             |
| ----- | --------- | -------------- | -------------- | -------------------------------- |
| P-001 | 대시보드  | 메인 대시보드  | `/`            | 핵심 지표, 오늘 경기, 공지 요약  |
| P-002 | 회원 관리 | 회원 목록      | `/members`     | 회원 검색/필터, 엑셀 내보내기 등 |
| P-003 | 회원 관리 | 회원 상세      | `/members/:id` | 상세 정보, 출석/정산 이력        |
| P-004 | 경기 관리 | 경기 일정 목록 | `/matches`     | 예정/완료 경기 리스트            |
| P-005 | 정산      | 회비/정산 현황 | `/billing`     | 회비 납부 현황, 미납 알림 등     |

### 3. 정보 구조 다이어그램(개략)

- **권장**: Notion / FigJam / Miro 등 외부 툴에서 아래 내용을 다이어그램으로 작성 후 링크 추가
  - 사용자 유형(관리자, 운영진, 일반 회원 등)
  - 각 사용자 유형별 접근 가능한 메뉴/페이지
  - 핵심 플로우(회원 가입 → 승인 → 정산 → 알림 등)

링크 예시:

- IA 다이어그램: `[링크를 여기에 삽입]`

### 4. 사용자 유형(Stakeholder) 정의

- **관리자(클럽 대표)**
  - 모든 메뉴 접근 가능
  - 요금제/결제, 클럽 설정 변경 가능
- **운영진(스태프)**
  - 회원 관리, 경기 관리, 일부 정산 메뉴 접근
- **일반 회원**
  - 본인 정보 조회, 일정 확인, 알림 수신

### 5. 데이터 관점의 IA 정리

- **핵심 엔티티**
  - 회원(Member)
  - 클럽(Club)
  - 경기(Match/Game)
  - 정산/회비(Billing/Payment)
  - 공지/알림(Notification)

### 6. 변경 이력

- IA 변경 시, 어떤 메뉴/페이지가 추가/삭제/병합되었는지 날짜와 함께 기록합니다.
