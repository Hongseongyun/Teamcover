## 03. 디자인 가이드

Teamcover 서비스의 **디자인 시스템 및 화면 시안**을 정리하는 문서입니다.

### 1. 디자인 시스템 개요

- **브랜드 키워드**: (예) 스포츠, 에너지, 신뢰, 직관
- **톤앤매너**: 밝고 경쾌한 분위기, 직관적이고 관리자가 쓰기 편한 UI

### 2. 컬러 시스템

| 용도      | 토큰                | 값(예시)  | 설명                       |
| --------- | ------------------- | --------- | -------------------------- |
| Primary   | `--color-primary`   | `#2563eb` | 주요 액션 버튼, 하이라이트 |
| Secondary | `--color-secondary` | `#0f172a` | 헤더/내비게이션            |
| Accent    | `--color-accent`    | `#f97316` | 포인트, 배지 등            |
| Success   | `--color-success`   | `#16a34a` | 성공/완료 상태             |
| Danger    | `--color-danger`    | `#dc2626` | 오류/경고 상태             |

> 실제 사용 중인 색상값은 `frontend/src/App.css`, `Navbar.css` 등을 참고해 추출 후 이 표를 업데이트하세요.

### 3. 타이포그래피

- **기본 폰트 패밀리**
  - 웹: `Pretendard`, `Noto Sans KR`, system UI 등
- **텍스트 스타일 예시**

| 스타일    | 예시 클래스  | 용도              |
| --------- | ------------ | ----------------- |
| Heading 1 | `.text-3xl`  | 페이지 타이틀     |
| Heading 2 | `.text-2xl`  | 섹션 타이틀       |
| Body      | `.text-base` | 일반 본문         |
| Caption   | `.text-xs`   | 부가 설명, 레이블 |

### 4. 공통 컴포넌트 디자인

- **버튼**
  - 기본, 비활성, 로딩, 위험(삭제) 상태 정의
  - 비율/여백/라운드 값 통일
- **폼 요소**
  - Input, Select, Checkbox, Radio 등 스타일 통일
  - 에러 메시지/유효성 피드백 디자인
- **카드/패널**
  - 대시보드용 카드, 리스트용 카드

> 실제 구현은 `frontend/src/components` 의 컴포넌트와 연결해서 관리하면 좋습니다.

### 5. 반응형(Responsive) 기준

- **브레이크포인트 예시**
  - 모바일: \< 640px
  - 태블릿: 640px ~ 1024px
  - 데스크톱: \> 1024px
- 각 구간별로 레이아웃/컴포넌트 크기 변경 규칙을 간단히 정리합니다.
